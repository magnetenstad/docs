Guidelines for taking notes

A note

  1. should represent one concept
  2. should be informative in isolation
  3. should link to other relevant notes

An article

  1. should build upon a set of notes
  2. should elaborate on the relation between notes

Folders

  1. should not be used

Files

  1. should be uniquely identified by names such as domain.subdomain.name_of_file.html

Attachments

Markdown

  1. should not use wikilinks for links and images
